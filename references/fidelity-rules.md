# 保真规则（Fidelity Rules）

本文件定义 paper-report 汇报稿的**高保真章节**处理规则。所有标记为"高保真"的章节必须严格遵守以下规则。

## 适用范围

以下章节类型为**高保真**，禁止任何形式的删减：

- Problem Definition / Task Formulation（问题定义与任务设定）
- Method / Approach（方法）
- Experiments — Setup & Main Results（实验设置与主结果）
- Ablation Study / Analysis（消融与分析）
- Limitations / Future Work（局限与未来工作）
- Related Work（相关工作，引言中已覆盖的部分除外）

---

## R1: 结构保真

- 保留原文的章节层级结构（section → subsection → subsubsection）
- 子标题使用原文标题的忠实翻译（附英文原文），如："3.2 跨模态融合（Cross-Modal Fusion）"
- 不合并、拆分或重排原文章节顺序
- 如原文无明确子标题但内容可区分，可添加描述性子标题并标注"[编者注]"

## R2: 术语保真

- 关键术语首次出现：采用"中文翻译（English Original）"格式
  - 示例：注意力机制（Attention Mechanism）
- 后续出现：可仅用中文翻译，但必须全篇一致
- 不可自行发明翻译——优先使用领域内公认译法；无公认译法时保留英文原文
- 缩写首次出现必须展开全称

## R3: 公式保真

- 完整保留原文所有编号公式的 LaTeX 表达式
- 不改写数学含义（不可将 $\sum$ 改为"求和"替代公式）
- 不省略变量定义——每个公式首次出现时，附简洁符号定义表：

```markdown
$$\mathcal{L} = \mathcal{L}_{cls} + \lambda \mathcal{L}_{reg}$$

| 符号 | 含义 |
|------|------|
| $\mathcal{L}$ | 总损失函数（Total Loss） |
| $\mathcal{L}_{cls}$ | 分类损失（Classification Loss） |
| $\mathcal{L}_{reg}$ | 正则化损失（Regularization Loss） |
| $\lambda$ | 正则化权重系数 |
```

- 公式编号与原文保持一致（如 Eq.(3)、Equation 3）
- 未编号但关键的等式也应保留，可添加标记如"[未编号]"

## R4: 图表保真

### 图片嵌入（硬约束）

- **禁止**使用任何文字占位符代替图片，包括但不限于：
  - `[Figure 1]`、`[图1]`、`fig1`、`（见图1）`
  - `[此处应有图片]`、`[图片省略]`
- **必须**使用 `extract_figures.py` 裁剪的实际 PNG 图片
- 嵌入格式：`![Fig.X: caption](./images/figure_xxx.png)`
- 图表在对应章节**就地嵌入**，紧跟原文讨论该图表的位置

### 图表结论

- 每张图/表下方附 1-2 句**结论方向总结**
- 总结来自原文对该图/表的讨论，非汇报者主观评价
- 格式：`> **图 X 要点：** xxxxxxx`

### 表格处理

- 简单表格：用 Markdown 表格复刻（保留原始数值精度）
- 复杂表格（多级表头、合并单元格）：使用裁剪的表格图片
- 数值精度：原文 76.3% 不可改为"约76%"或"76%"

## R5: 证据追溯

- 每个核心结论必须可追溯到原文来源
- 追溯锚点格式：`（§X.Y, Table Z）` 或 `（见图 N）`
- 方法章节：每个设计决策附原文给出的理由
- 实验章节：每个数值结论标明来源表格/图表

## R6: 禁止教学化改写

以下行为在高保真章节中**严格禁止**：

| 禁止 | 替代做法 |
|------|---------|
| 使用生活类比解释技术概念 | 用原文术语直接表述 |
| "大白话"改写核心方法 | 保留原文的技术表述 |
| "你可以把它理解为…" | 直接给出定义 |
| "简单来说就是…" | 不做简化，保留完整表述 |
| "想象一下…" | 不使用想象类引导 |
| 添加主观评价（"很巧妙"、"有趣的是"） | 客观陈述原文内容 |
| 省略"看起来不重要"的细节 | 保留原文提到的所有技术细节 |

**唯一例外**：如某段原文确实晦涩且影响汇报理解，可在段末添加 `[编者注: ...]` 简要澄清，但不可替代原文表述。

## 校验清单

在生成汇报稿后，逐条检查：

- [ ] 高保真章节是否存在未经授权的删减？
- [ ] 所有图表是否使用实际 PNG 嵌入（无文字占位符）？
- [ ] 公式编号是否与原文一致？
- [ ] 关键术语是否全篇使用一致的翻译？
- [ ] 数值结果是否保留原文精度？
- [ ] 每个核心结论是否有来源锚点？
- [ ] 是否存在教学化改写痕迹？
